# 资源管理系统 - 快速开始指南

## 5分钟快速上手

### 1. 启动项目

```bash
# 启动设计端（推荐）
npm run dev:designer
```

访问: http://localhost:5173

页面会自动跳转到资源管理界面。

### 2. 基本操作

#### 查看资源列表

- 页面加载后自动显示资源列表
- 表格展示所有资源信息

#### 搜索资源

1. 在搜索框输入关键词
2. 点击"搜索"按钮或按回车键
3. 表格自动刷新显示结果

#### 新建资源

1. 点击"新建资源"按钮
2. 填写表单信息
3. 点击"确定"提交

#### 编辑资源

1. 点击表格中的"编辑"按钮
2. 修改表单信息
3. 点击"确定"保存

#### 删除资源

1. 点击表格中的"删除"按钮
2. 确认删除操作
3. 资源被删除

## 功能说明

### 表单字段说明

| 字段     | 必填 | 说明                                       |
| -------- | ---- | ------------------------------------------ |
| 父级资源 | 否   | 选择父级资源，不选则为根节点               |
| 资源名称 | 是   | 资源的显示名称                             |
| 菜单编码 | 是   | 唯一标识，只能包含字母、数字、下划线、横线 |
| 业务模块 | 是   | 所属的业务模块                             |
| 节点类型 | 是   | 文件夹/页面/按钮                           |
| 排序序号 | 否   | 用于排序，数字越小越靠前                   |
| 路由路径 | 否   | 页面的路由路径                             |
| 图标     | 否   | 图标名称                                   |
| URL地址  | 否   | 外部链接地址                               |
| 元数据   | 否   | JSON格式的额外数据                         |

### 节点类型说明

**文件夹 (1)**:

- 用于组织菜单结构
- 可以包含子节点
- 不能直接访问

**页面 (2)**:

- 实际的页面路由
- 可以配置路由路径
- 可以直接访问

**按钮 (3)**:

- 页面内的操作按钮
- 用于权限控制
- 不显示在菜单中

## 常见问题

### Q1: 为什么看不到数据？

**A**: 需要后端API支持。确保：

1. 后端服务已启动
2. API地址配置正确
3. 网络连接正常

### Q2: 如何修改API地址？

**A**: 修改环境变量文件：

```bash
# envs/.env.designer.dev
VITE_SERVICE_URL=http://your-api-url
```

### Q3: 新建资源后看不到？

**A**: 点击"刷新"按钮重新加载数据。

### Q4: 删除资源失败？

**A**: 可能原因：

1. 资源有子节点（需要先删除子节点）
2. 资源被其他地方引用
3. 没有删除权限

### Q5: 如何切换到设计器？

**A**: 在浏览器地址栏输入: http://localhost:5173/designer

## 性能建议

### 大数据量优化

- 使用分页，每页不超过100条
- 使用搜索筛选缩小数据范围
- 避免一次性加载所有数据

### 网络优化

- 使用CDN加速静态资源
- 启用Gzip压缩
- 使用HTTP/2

## 故障排除

### 问题: 页面空白

**解决**:

1. 检查浏览器控制台错误
2. 检查网络请求是否正常
3. 清除浏览器缓存

### 问题: 路由跳转失败

**解决**:

1. 检查路由配置
2. 检查路由守卫
3. 检查认证状态

### 问题: 表单提交失败

**解决**:

1. 检查表单验证
2. 检查API请求
3. 查看错误提示
